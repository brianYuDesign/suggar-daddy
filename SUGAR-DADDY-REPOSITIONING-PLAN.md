**開始日期**: 2026-02-19 09:21 GMT+8

---

## 🎯 戰略方向 & 目標

### 現在的定位（過於寬泛）
- 「約會 + 訂閱/內容平台」
- 不清楚是主打約會還是主打內容
- 用戶心智混亂

### 新定位（焦點清晰）
**「內容創作者的社交商務平台」**

- 🎬 **核心**: OnlyFans 的內容訂閱（核心收入）
- 💬 **社交**: Tinder 的匹配推薦（用戶留存）
- 💳 **支付**: 已有的完整支付系統（優勢）

### 目標用戶轉變

| 面向 | 現在 | 新方向 |
|------|------|--------|
| **主要用戶** | 求偶者 | 內容創作者 |
| **次要用戶** | 訂閱者 | 內容粉絲 |
| **收入來源** | 約會配對 | 內容訂閱 (60%) + 打賞 (30%) + 匹配推薦 (10%) |
| **用戶旅程** | 配對 → 聊天 | 發現 → 訂閱 → 互動 |

## 📊 產品框架重新設計

### 核心三層結構

```
層級 1: 創作者中心 (OnlyFans 模式)
├─ 內容發佈 (文字、圖片、視頻、直播)
├─ 訂閱管理 (免費 + 付費層級)
├─ 粉絲互動 (評論、DM、打賞)
└─ 收入儀表板 (提現、收入分析)

層級 2: 內容發現 (Tinder 模式)
├─ 推薦引擎 (基於興趣匹配)
├─ 卡片化 UI (垂直滑動發現)
├─ 一鍵訂閱
└─ 快速配對 (可選的社交匹配)

層級 3: 商務系統 (已有的基礎)
├─ 訂閱計費
├─ 支付結算
├─ 合規報稅
└─ 數據分析
```

### 用戶旅程重新定義

**創作者**:
```
1. 註冊 → 驗證身份
2. 建立主頁 (頭像、簡介、定價)
3. 發佈內容 (第 1 天 3+ 内容吸引訂閱者)
4. 設定訂閱層級 (免費/¥99/¥199)
5. 粉絲開始來 (推薦系統帶量)
6. 互動 + 提現 (月底結算)
```

**粉絲**:
```
1. 註冊
2. 發現創作者 (推薦卡片、搜索)
3. 試看免費內容
4. 訂閱付費內容 (¥99-¥999)
5. 留言、打賞、互動
6. 可選: 配對互動 (Tinder 社交)
```

### 需要新增的 Services

| Service | 功能 | 優先級 | 預計工期 |
|---------|------|--------|---------|
| **Content-Streaming** | 視頻直播、流媒體 | P0 | 3-4 週 |

| **Creator-Analytics** | 創作者數據分析 | P1 | 2 週 |
| **Notification** | 推送通知系統 | P1 | 1-2 週 |
| **Moderation** | 內容審核系統 | P0 | 2 週 |
| **Report-Compliance** | 報稅合規系統 | P2 | 1-2 週 |

### 需要改造的 Services

| Service | 改動 | 理由 | 工期 |
|---------|------|------|------|
| **User-Service** | 加創作者檔案 | 區分創作者/粉絲 | 1 週 |
| **Content-Service** | 支持訂閱內容鎖定 | 付費內容保護 | 1-2 週 |
| **Payment-Service** | 加分成計算 | 創作者提現 | 1-2 週 |
| **Subscription-Service** | 層級訂閱管理 | 多層定價 | 1 週 |
| **Media-Service** | 支持大文件 + CDN | 視頻優化傳輸 | 2 週 |
| **Skill-Service** | 改名: Creator-Profile | 更清楚語義 | <1 週 |

### 架構圖變化

**現在（亂）**:
```
User ↔ Content ↔ Media
  ↓       ↓
Payment Subscription
```

**新（清晰）**:
```
         Recommendation Engine
              ↓
        (推薦卡片發現)
         ↓       ↓
    Creator  →  Fan
    發佈內容    訂閱
         ↓
    Creator-Analytics
    收入追蹤
         ↓
    Payment (分成計算)
```

## ⚠️ 風險 & 緩解

| 風險 | 可能性 | 影響 | 緩解方案 |
|------|--------|------|---------|
| **推薦算法效果不佳** | 中 | 新用戶留存低 | 先用簡單策略，逐步優化 |
| **創作者審核耗時** | 高 | 新用戶體驗差 | 建立自動化審核 + 人工抽查 |
| **內容合規風險** | 中 | 法律問題 | AI 審核 + 用戶舉報機制 |
| **支付結算延遲** | 低 | 創作者不滿 | 提前對接銀行、自動提現 |
| **技術債務堆積** | 高 | 後期難以擴展 | 重點代碼審查、測試優先 |
