## 使用說明

### 填寫指南

1. **執行摘要**：簡潔明瞭，適合快速閱讀（管理層友好）
2. **完成項目**：詳細記錄成果和亮點
3. **進行中項目**：誠實報告進度和困難
4. **指標追踪**：數據驅動，展示趨勢
5. **風險管理**：前置識別，主動應對
6. **學到的經驗**：團隊學習和成長

### 時間安排

- **撰寫時間**：每週五下午 14:00-16:00
- **審查時間**：週五 16:00-17:00（團隊 Sprint Review）
- **發送時間**：週五 17:00（給管理層和團隊）

### 受眾

- **主要受眾**：Tech Lead, 管理層
- **次要受眾**：全體開發團隊
- **格式**：Markdown（便於版本控制）
